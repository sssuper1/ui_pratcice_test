# 基于 Qt 的网管上位机软件实现指南

结合本项目的应用层协议（直接管理电台后端的 `mgmt-ctrl`），如果需要使用 Qt（C++）从零实现一套能够**实时展示网络拓扑**并能**下发修改设备参数**的桌面网管软件，需要按照以下几个核心模块来构建。

---

## 总体架构设计

在 Qt 中，建议采用经典的 **MVC（模型-视图-控制器）** 架构或 **信号槽事件驱动** 模式。
主要包含三大模块：
1. **协议层 (Protocol)**：精确映射 C 语言结构体，处理大小端（Endian）转换和字节对齐。
2. **通信层 (Network)**：使用 `QUdpSocket` 处理与 `16000`、`7600` 等电台核心端口的异步通信。
3. **表现层 (UI/View)**：使用 `QGraphicsView` 绘制拓扑关系图，使用 `QTableView` 展现实时链路状态（SNR、RSSI），使用表单页面捕获参数修改。

---

## 先回答你的问题：上位机暴露接口是什么，是否网线直连

是的，当前工程是通过 **网线直连电台 `eth0`** 与上位机网管软件做信息传输。软件层暴露了两套并行接口：

1. **网管控制接口（Smgmt 协议）**
    - 协议头：`Smgmt_header`（`mgmt_head=0x4C4A`，网络序）
    - 典型端口：`16000`（登录/地面口）、`7600`（组网控制主口）、`7601`（TCP升级）
    - 典型命令：`MGMT_LOGIN(0x0000)`、`MGMT_SET_PARAM(0x4d02)`、`MGMT_TOPOLOGY_REQUEST(0x4d0e)`、`MGMT_TOPOLOGY_INFO(0x3c03)`

2. **业务上位机接口（APP_HEAD 协议）**
    - 协议头：`APP_HEAD`（`head=240`）
    - 端口：`QK_WG_PORT=10000`
    - 方向：
      - 上位机 -> 电台：`CMD_DEV_CONFIG(42)`、`CMD_CH1_CONFIG(43)`
      - 电台 -> 上位机：`CMD_CONFIG_ACK(44)`、`CMD_NET_STATUS_REPORT(41)`、`CMD_METRICS_REPORT(45)`、`CMD_SELF_TEST_INFO(46)`、`CMD_DEV_STATUS_INFO(47)`、`CMD_CH1_STATUS_INFO(48)`

> 工程要点：即使你要给深空节点下发参数，物理入口仍是你网线直连的那台电台（网关节点），它在 `7600` 口执行转发。

## 阶段一：协议结构体的 Qt C++ 映射与对齐

这是最容易发生“发包电台不认”或者“收包解析乱码”的地方。Qt 默认会按照 4 字节对齐，而网络包经常是紧凑排列的，因此需要使用 `#pragma pack(1)`。在 Qt 项目的头文件中定义：

```cpp
#include <cstdint>
#include <QtGlobal>

#pragma pack(push, 1)

// 统一外层信封
struct Smgmt_header {
    quint16 mgmt_head; // 固定头部标识, 当前工程为 0x4C4A
    quint16 mgmt_len;  // 后续 mgmt_data 的长度
    quint16 mgmt_type; // 报文类型（如 0x4d02 设参，0x3c03 拓扑上报）
    quint16 mgmt_keep; // 保留字段或特殊位图
    // quint8 mgmt_data[0]; // 变长数据区
};

// 本机设参结构体 (Smgmt_set_param 缩略版)
struct Smgmt_set_param {
    quint16 mgmt_id;              // 节点ID
    quint16 mgmt_route_interval;  // 路由间隔
    // ... 其他字段 ...
    quint32 mgmt_mac_freq;        // 频率
    quint16 mgmt_mac_txpower;     // 发射功率
    quint8  mgmt_mac_bw;          // 带宽
    // ...
};

// 组网设参归一化结构体（跨节点转发设参）
struct Smgmt_param {
    quint32 mgmt_ip;       // 目标设备IP
    quint16 mgmt_id;       // 目标设备ID
    // ... 其余参数（与上面类似）
};

#pragma pack(pop)
```

## 阶段一补充：APP_HEAD 业务协议（端口 10000）

如果你的上位机对接的是 `mgmt_recv_from_qkwg/thread_report_test` 线程，而不是 `mgmt_recv_msg` 的 Smgmt 主链，那么需要使用 `APP_HEAD`。

```cpp
#pragma pack(push, 1)

struct APP_HEAD {
    quint8  head;        // 固定 240
    quint8  len;         // 头长度
    quint16 info_len;    // 头+正文总长度
    quint8  packet_type; // 42/43/44/41/45/46/47/48...
    quint8  activity_type;
    quint16 send_type;
    quint16 send_id;
    quint32 timestamp_1;
    quint16 timestamp_2;
    quint8  packet_type_2;
    quint8  flag;
    quint16 recv_type;
    quint16 recv_id;
    quint32 seq;
    quint32 data_len;
};

#pragma pack(pop)
```

实现建议：Qt 侧将 `APP_HEAD` 固定作为前导头，`packet_type` 决定后续 payload 结构体解释方式。

## 阶段二：建立 UDP 通信中枢并执行“接管三步走”

在 Qt 中，建立一个 `NetworkManager` 类继承自 `QObject`，它拥有一个 `QUdpSocket`。

```cpp
#include <QUdpSocket>

class NetworkManager : public QObject {
    Q_OBJECT
public:
    NetworkManager() {
        udpSocket = new QUdpSocket(this);
        // 绑定本地任意端口，用于接收电台回传的包
        udpSocket->bind(QHostAddress::AnyIPv4, 0); 
        connect(udpSocket, &QUdpSocket::readyRead, this, &NetworkManager::processPendingDatagrams);
    }

    void startTakeover(const QString& radioIp) {
        QHostAddress target(radioIp);
        
        // 1. 发送 LOGIN 宣告接管 (目标端口 16000)
        Smgmt_header loginMsg;
        loginMsg.mgmt_head = qToBigEndian<quint16>(0x4C4A);
        loginMsg.mgmt_type = qToBigEndian<quint16>(0x0000); // MGMT_LOGIN
        loginMsg.mgmt_len = 0;
        udpSocket->writeDatagram((const char*)&loginMsg, sizeof(loginMsg), target, 16000);

        // 2. 发送 TOPOLOGY_REQUEST 抢夺状态上报网关 (目标端口 7600)
        // 这个包非常关键，发出去之后，电台就会把当前电脑的 IP:本地端口 设为 gate_addr
        Smgmt_header topoReq;
        topoReq.mgmt_head = qToBigEndian<quint16>(0x4C4A);
        topoReq.mgmt_type = qToBigEndian<quint16>(0x4d0e); // MGMT_TOPOLOGY_REQUEST
        topoReq.mgmt_len = qToBigEndian<quint16>(4);
        
        QByteArray packet;
        packet.append((const char*)&topoReq, sizeof(topoReq));
        quint32 myIp = /* 获取本机IP并转网络序 */;
        packet.append((const char*)&myIp, 4);
        
        udpSocket->writeDatagram(packet, target, 7600);
    }

private slots:
    void processPendingDatagrams();

private:
    QUdpSocket *udpSocket;
};
```

## 阶段三：解析电台主动上报的报文（实时展现更新）

一旦你的桌面程序接管成功，电台会在后台依靠 `mgmt_get_msg` 每 5 秒主动通过 `send_topo_msg` 向你的软件吐出一次 `0x3c03` (MGMT_TOPOLOGY_INFO) 数据流。

在刚才的 `processPendingDatagrams` 槽函数中处理：

```cpp
void NetworkManager::processPendingDatagrams() {
    while (udpSocket->hasPendingDatagrams()) {
        QByteArray datagram;
        datagram.resize(udpSocket->pendingDatagramSize());
        udpSocket->readDatagram(datagram.data(), datagram.size());

        if (datagram.size() < sizeof(Smgmt_header)) continue;

        Smgmt_header* header = (Smgmt_header*)datagram.data();
        quint16 type = qFromBigEndian(header->mgmt_type);

        if (type == 0x3c03) { // 拓扑和实时链路质量数据
            int offset = sizeof(Smgmt_header);
            
            // 提取本端指标 (提取从网线直连的那台电台的本端属性)
            quint32 selfIp = qFromBigEndian(*(quint32*)(datagram.data() + offset));
            offset += 4;
            // 跳过经纬度底噪等...直达 neighbors_num
            offset += (8 + 8 + 4 + 4 + 4); 
            quint32 neighbors_num = qFromBigEndian(*(quint32*)(datagram.data() + offset));
            offset += 4;

            // 循环提取邻居数据并发射信号通知 UI
            for (quint32 i = 0; i < neighbors_num; ++i) {
                // 读取各个指标：ID, IP, 时延, SNR, RSSI
                // 触发信号：emit updateLinkQuality(selfIp, neighborIp, snr, rssi);
                offset += 12; // 假设每条12字节
            }
        }
    }
}
```

* **UI 联动**：在 Qt 主界面中，接收到 `updateLinkQuality` 信号后，更新 `QTableView` 的底层 Model，界面上的折线图 (`QtCharts`) 或表格立刻跳动。

## 阶段四：参数修改与下发实现（点亮电台修改）

UI 界面抓取用户输入的设参数据（例如选择了“修改发射功率为 30dBm”），我们需拼装出 `MGMT_SET_PARAM` (点对点设参) 并将其发射向 7600 端口。

```cpp
void NetworkManager::setParameter(const QString& targetIpStr, quint32 newFreq, quint8 newBw) {
    QByteArray packet;
    
    // 1. 组装信封 Header
    Smgmt_header header;
    header.mgmt_head = qToBigEndian<quint16>(0x4C4A);
    header.mgmt_type = qToBigEndian<quint16>(0x4d02); // MGMT_SET_PARAM
    // 长度 = mgmt_data(4字节IP + Smgmt_param的大小)
    header.mgmt_len = qToBigEndian<quint16>(4 + sizeof(Smgmt_param)); 
    // keep 可能需要根据位图标识哪些字段生效
    header.mgmt_keep = qToBigEndian<quint16>( /* 根据需要设置有效位掩码 */ ); 
    
    packet.append((const char*)&header, sizeof(header));

    // 2. 写入目标 IP（前缀 4 字节，用于电台软路由判定是否自己消化）
    quint32 dstIp = QHostAddress(targetIpStr).toIPv4Address(); // 本地序
    quint32 netDstIp = qToBigEndian(dstIp); // 网络序
    packet.append((const char*)&netDstIp, 4);

    // 3. 填充组网参数载荷 Smgmt_param
    Smgmt_param param;
    memset(&param, 0, sizeof(param)); // 初始化清零
    
    param.mgmt_ip = netDstIp; 
    param.mgmt_mac_freq = qToBigEndian(newFreq);
    param.mgmt_mac_bw = newBw; // 单字节无需转换
    
    packet.append((const char*)&param, sizeof(param));

    // 4. 将包打给定点接入的那个物理电台IP
    // 注意：哪怕我是给深空的一台 192.168.2.5 下发配置，
    // 该UDP包依然是用直连我的网线电台 192.168.2.1 作为物理转发入口！
    QHostAddress gatewayRadioIp("192.168.2.1"); 
    udpSocket->writeDatagram(packet, gatewayRadioIp, 7600);
}
```

## 阶段四补充：若采用 APP_HEAD（10000 端口）如何下发

若你选择直接对接上位机线程 `mgmt_recv_from_qkwg`，下发入口不是 `Smgmt_header`，而是：

1. `APP_HEAD(packet_type=42)` + `DEVICE_PARAM_SET`（设备级参数）
2. `APP_HEAD(packet_type=43)` + `CHANNEL_PARAM_SET`（信道级参数）

电台收到后会：

- 执行参数落地（含 `/etc/node_xwg` 同步、`mgmt_netlink_set_param` 下发）
- 回 ACK：`APP_HEAD(packet_type=44)` + `PARAM_SET_ACK`
- 持续周期上报状态：`41/45/46/47/48`

## 阶段五：推荐落地流程（按真实代码）

这里给一套可以直接联调的实现流程，按“先连通、再接管、再展示、再下发”的顺序：

1. **网线直连并配置同网段**
    - 上位机网卡配置到 `192.168.2.x/24`，直连目标电台 `eth0`。

2. **选择协议通道**
    - 要做 Mesh 拓扑 + 跨节点参数：优先走 `Smgmt_header`（16000/7600）。
    - 要对接 20 所业务上位机链路：走 `APP_HEAD`（10000）。

3. **接管与订阅（Smgmt）**
    - 先发 `MGMT_LOGIN` 到 `16000`。
    - 再发 `MGMT_TOPOLOGY_REQUEST` 到 `7600`。
    - 监听 `MGMT_TOPOLOGY_INFO(0x3c03)`，解析邻居与链路指标。

4. **参数下发**
    - Smgmt 路径：发 `MGMT_SET_PARAM(0x4d02)` 到你直连的网关电台 `7600`。
    - APP_HEAD 路径：发 `packet_type=42/43` 到 `10000`，等待 `44` ACK。

5. **UI 联动展示**
    - 拓扑图：按 `selfip + neighbors[]` 更新节点与边。
    - 参数页：按 ACK/回传状态更新控件与日志。
    - 建议做差量刷新，避免 5 秒周期全量重绘闪烁。

## 阶段六：Qt 界面设计指引

在 Qt Creator (或 Qt Designer) 中设计：

1. **左侧拓扑视图 (`QGraphicsView`)**：
   * 将每个接收到的 IP 实例化为一个 `QGraphicsPixmapItem`（设备图标）。
   * 使用 `QGraphicsLineItem` 连接有 `neighbor_num` 关系的设备。
   * 在连线上吸附 `QGraphicsTextItem` 显示 SNR/时延。绿色代表优，红色代表差。
2. **右侧参数控制面板 (`QDockWidget` + `QFormLayout`)**：
   * 在拓扑图中选中某个节点图标时，右侧更新为对应的设置面板（从缓存的设备信息中反显当前的频率、频宽）。
   * 添加 【应用设置】 按钮，点击即拦截用户表单数据，调用 `setParameter`。
3. **底部日志面板 (`QTextBrowser`)**：
   * 提供报文发收明细。例如抓取每次 Socket 提取到的十六进制流转化为易读文本并打印出时间戳（类似串口助手体验），方便联调底层。

### 注意事项：
* **大小端雷区**：通常 ARM Linux (i.MX6ULL) 和 x86 电脑端存在大小端差异。在网络收发时，一定不要偷懒直接传结构体拷贝，务必逐个字段地执行 `qToBigEndian` (也就是 `htonl` 的 Qt 版本) 和 `qFromBigEndian`。
* **信号防抖**：拓扑报文每 5 秒上报可能会带来 UI 全量刷新闪烁。建议在 Qt Model 中比对数据哈希或差异，仅更新变化过的字段数据（例如当 `new_snr != old_snr` 时才触发 `dataChanged`）。
